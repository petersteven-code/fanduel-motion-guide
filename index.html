fd.kick · 340ms · ease-out aggressive
The sheet rises fast from below and decelerates into place — fd.kick's long tail means it arrives with authority, not a bounce. The backdrop fades 80ms faster than the sheet so context is always visible.

Input focus
fd.snap · instant border highlight
Border colour transition is 80ms linear — micro-interactions should feel immediate. The keyboard appearing is the main event; the border is supporting signal. Never animate focus states with a slow ease.
borderProgress.value = withTiming(1, {
duration: 80,
easing: Easing.linear

// borderProgress.value,

"Place Bet" press
fd.snap press-in · fd.release on confirm
Press-in: spring snap to 0.96 (anticipation). Release: spring back to 1. The snap happens in the button; the confirmation state appears in the sheet — two separate animations that feel like one continuous action.
pressIn: scale → 0.96, spring s:500 d:38
pressOut: scale → 1, spring s:300 d:20
then: trigger confirmation overlay
4
Confirmation overlay
fd.release · 440ms · earned expressive
The success state bounces in with overshoot — the first moment of expressiveness in this flow. The checkmark has its own spring delay (80ms), and the ring expands outward as secondary action. Three elements, one feeling.
card: scale(0.8)→1, bezier(0.34,1.45,0.64,1)
check: scale(0)→1, 80ms delay, same curve
ring: scale(1)→2.2, opacity 1→0, 500ms ease-out
5
Sheet dismisses
ease-in · 220ms · always faster than enter
Exit is always faster than enter — 220ms vs 340ms. Ease-in (accelerating) mirrors how objects naturally fall away. Backdrop fades in 180ms, concurrent. Swipe-down dismissal passes gesture velocity to spring.

easing: Easing.in(Easing.cubic)

Win amount counts up
interpolate · 700ms · ease-out cubic
The number counts up from $0 to the win amount — building anticipation before the reveal. Ease-out means it slows as it approaches the final number, adding suspense at the end. Starts 300ms after the card enters.
interpolate(progress, [0,1], [0, WIN_AMOUNT])

Confetti
ease-in fall · staggered 0–500ms
~40 particles, each with a random start delay (0–500ms) so it rains, not floods. Ease-in on the Y-axis simulates gravity — they accelerate as they fall. Random rotation adds life. Keep it to 1 second total duration per piece.

duration: 900 + Math.random() * 600,

})

The CTA appears last — after the emotion has landed. If it appeared simultaneously with the win card, it would feel transactional. Waiting 650ms means the user has felt the win before being asked to do anything about it.

Leg count badge
fd.snap spring · s:500 d:38 · scale pulse
The badge bumps scale briefly on each new leg. Spring physics mean it settles without any explicit return animation. This makes the count feel reactive, not mechanical — like a scoreboard updating.
withSpring(scale, {

Combined odds counter
fd.snap · instant flash + 300ms fade
Odds update instantly — no count-up here, because the user needs to trust the number is accurate. A brief green color flash signals "this changed" and fades over 300ms. The number itself never animates.
// Flash color signal

}) // → return to default

ease-out cubic · 500ms delay:100ms

// JS interpolate, not RN
const ease = 1 - Math.pow(1-t, 3);
current = ease * targetPayout;
// 500ms, 100ms delay

withTiming(opacity, { duration: 200 })

Loading States

Three distinct patterns — never interchangeable. Skeleton for content-heavy surfaces (lobby, game detail). Shimmer signals scanning in progress. Spinner only for indeterminate actions with no predictable end. IBM Carbon and Material Design 3 both define this taxonomy. The mistake most apps make: using a spinner where a skeleton would be faster-feeling.

Skeleton structure
pulse · 1.4s ease-in-out · infinite alternate
Skeleton blocks match the exact dimensions of real content. The pulse keeps opacity between 100% and 30% — never disappearing, so the shape stays readable. Chrome (nav, tab bar) never skeletons — only content areas do. Reference: IBM Carbon "skeleton states."

), -1, true // reverse: true
)

Shimmer scan
linear · 1.2s · translateX(-100%→100%) · infinite
A highlight sweeps left-to-right over the skeleton, simulating active scanning. Linear easing only — any easing on shimmer looks wrong. Applied to content-heavy cards (hero, game rows). Do not apply to every skeleton — too much shimmer reads as unstable.
translateX.value = withRepeat(
withTiming(width, {
duration: 1200,
easing: Easing.linear
}), -1, false
)
// Start value: -width (off left)
3
Skeleton → content crossfade
fd.kick · 280ms · opacity + stagger 40ms
When data arrives, stop the shimmer immediately and crossfade skeleton → real content. Opacity only — never scale or translate. The skeleton disappears as the content appears, staggered top-to-bottom. Structural positions never shift — this prevents layout jump.
// Skeleton out
skelOpacity.value = withTiming(0, { duration: 200 })
// Content in, staggered
contentOpacity.value = withDelay(index * 40,
withTiming(1, { duration: 280,
easing: Easing.bezier(0, 0, 0.06, 1) })
)
4
Spinner — when to use
rotate · linear · 800ms · infinite
Spinner is reserved for actions with no predictable layout — placing a bet, processing payment, submitting a form. Never use for page loads (use skeleton). Never use alongside a skeleton. One rotation = 800ms linear. Size: 20px inline, 32px full-screen.
rotate.value = withRepeat(
withTiming(360, {
duration: 800,
easing: Easing.linear
}), -1, false
)
// Never ease — linear only for spinners
Patterns · 02

Reduced Motion

WCAG 2.1 AA requires it. Vestibular disorders affect 35% of adults over 40 — large-scale motion and parallax can cause genuine physical discomfort. prefers-reduced-motion is the OS signal, but FanDuel should also expose a manual toggle in Accessibility settings. The rule: remove translate/scale animations entirely. Preserve opacity transitions at half duration — they are safe.

1
What to remove
translateX / translateY / scale / rotate
In reduced motion mode: eliminate all translate and scale animations. These are the motion types that trigger vestibular symptoms. Shared element transitions collapse to crossfades. Screen transitions collapse to opacity only. No exceptions.
const isReduced = useReducedMotion()
// Instead of slide:
opacity.value = withTiming(1, {
duration: isReduced ? 150 : 340,
easing: Easing.linear
})
// translateY stays at 0 always
2
What to keep
opacity · half duration · linear
Opacity transitions are safe — they don't trigger vestibular response. Keep them at half the standard duration (e.g., 340ms → 170ms) so the UI still feels responsive. Live odds flashes keep their colour signal — colour change is not motion.
// Odds flash: safe in reduced motion
// Colour change ≠ motion
interpolateColor(flash.value,
[0, 1],
['transparent', 'rgba(0,200,122,.25)']
)
// Duration halved, not zero
3
Implementation
useReducedMotion() hook · OS + manual toggle
Reanimated exposes useReducedMotion() which reads the OS accessibility setting. FanDuel should also store a manual override in user preferences — some users want reduced motion only in the app, not system-wide. The manual toggle takes precedence.

Patterns · 03

Shared Element Transition

When a game card expands to the full detail screen, the card should travel — not disappear and reappear. The element is continuous across two screens; the transition makes that continuity visible. Apple calls this "hero animation." Material Design 3 calls it "container transform." Either way: the geometry interpolates from origin position to destination, while content inside cross-fades. This is the highest-effort, highest-return animation in the app.

Patterns · 04

Error & Feedback States

Every action has three possible outcomes: success, error, and warning. Success is covered in the Bet Slip screen. Error needs its own motion language — a rejection gesture that communicates "this didn't work" without alarming the user. The shake pattern (from Apple HIG) is the most recognisable. Carbon adds a "critical" vs "warning" distinction that FanDuel needs: failed bet placement is critical; changed odds is a warning.

Horizontal shake: the universal rejection signal from Apple HIG. 6 steps at 50ms each — totalling 300ms. Amplitude starts at ±8px, halves each pass so it decays naturally. The CTA button turns red simultaneously. Never use vertical shake — it reads as approval in some cultures.

withSpring(0, { stiffness: 500, damping: 38 })

The error message slides in from above as the shake plays out — 80ms delay so the shake registers first. fd.kick (aggressive ease-out) so it arrives with authority. Error messages should feel assertive, not apologetic. Auto-dismiss after 4 seconds with a gentle ease-in exit.

gold flash · fd.release on new value · no shake
Odds changing mid-slip is a warning, not an error — it requires a decision, not a rejection. The odds value flashes gold and bounces in with fd.release (slight overshoot), drawing attention without panic. The warning banner slides in below. The user must actively accept or decline the new odds.

Patterns · 05

Gesture Continuity

While a gesture is in progress, the UI follows the finger at exactly 1:1 — no easing, no lag. Physics engage only when the finger lifts. Apple HIG is unambiguous: any drag that doesn't track at 1:1 feels broken. On release, the element snaps to the nearest destination using spring with the gesture's exit velocity carried forward. This is what separates native-feeling apps from web-feeling apps.

1
1:1 drag tracking
no easing · direct value assignment
While the finger is down, translateY equals gesture translation directly — no animation function, no easing. This is the hardest rule to break correctly. Any lag or easing during active gesture makes the UI feel detached from the finger.
// In gesture handler (Pan Gesture)
onUpdate: (event) => {
// Direct assignment — no withTiming
translateY.value =
Math.max(0, event.translationY)
// Resist above snap point
}
2
Velocity handoff to spring
withSpring · velocity: event.velocityY
On finger lift, pass the gesture's exit velocity to withSpring. This creates physical continuity — a fast swipe snaps away instantly; a slow drag settles gently. Without velocity handoff the animation always feels the same regardless of how forcefully the user swiped.
onEnd: (event) => {
const shouldDismiss =
event.translationY > 100 ||
event.velocityY > 800
if (shouldDismiss) {
translateY.value = withSpring(
SHEET_HEIGHT,
{ velocity: event.velocityY }
)
} else {
translateY.value = withSpring(0)
}
}
3
Rubber-band resistance
Math.sqrt() decay · above snap point
If the user drags upward past the sheet's fully-open state, resistance kicks in — the sheet follows but decelerates, like pulling a rubber band. Square root decay is the physics formula Apple uses. Prevents the sheet from travelling off-screen while still feeling responsive.
// Rubber-band formula
const rubberBand = (x, max) => {
const c = 0.55 // resistance coefficient
return (1 - 1 / (x * c / max + 1)) * max
}
// Apply when dragging above snap:
translateY.value =
-rubberBand(Math.abs(overDrag), 80)

Empty States

Empty states have personality — but only the right kind. A first-time "My Bets" screen is an invitation: animate it in warmly. An empty search result is neutral: appear quietly. An empty state from a destructive action (bet expired, promo claimed) is loss: animate it in with even less fanfare than a loss state. Shopify Polaris and Atlassian both define these three distinct emotional tones for empty states.

A first-time empty My Bets screen is an opportunity to delight. The icon bounces in with fd.release (slight overshoot), then text and CTA stagger in below. The CTA block is animated last — after the context has landed. This is the highest expressiveness level appropriate for an empty state.

fd.kick · 240ms · opacity + translateY · no bounce
A no-results empty state appears after a search or filter action. No bounce — the user is already frustrated, don't add theatrical motion. Simple fade+slide in on fd.kick. Elements animate as a unit (not staggered) to feel quiet. Duration shorter than first-time empty.
// Everything as one unit

Expired / cleared — quiet
opacity only · 200ms · linear · no translate
When bets expire or a promo is claimed, the empty state appears after content leaves. Opacity crossfade only — no translate, no scale. This is the quietest empty state: the content has gone, now the absence is simply acknowledged. Respect the emotional context.
// Crossfade: content → empty
contentOpacity.value = withTiming(0, {
duration: 180,
easing: Easing.in(Easing.quad)
})
emptyOpacity.value = withDelay(100,
withTiming(1, {
duration: 200,
easing: Easing.linear

import { Easing } from 'react-native-reanimated';

// ── Curves ──────────────────────────────────────────────────────
export const curve = {
  snap:    { stiffness: 500, damping: 38 },           // spring · buttons, toggles
  kick:    Easing.bezier(0.0, 0.0, 0.06, 1.0),        // enter screen
  drive:   Easing.bezier(0.4, 0.0, 0.10, 1.0),        // A→B within screen
  release: Easing.bezier(0.34, 1.45, 0.64, 1.0),      // success, confirm
  crowd:   Easing.bezier(0.34, 1.8,  0.64, 1.0),       // win, peak moments
  rebound: { stiffness: 220, damping: 7 },            // spring · expressive oscillation
  exit:    Easing.in(Easing.cubic),                     // any element leaving screen
};

// ── Duration ────────────────────────────────────────────────────
export const dur = {
  micro:   80,    // button press-in, focus highlight, odds flash
  fast:    150,   // tooltips, chips
  base:    250,   // toasts, tab content fade
  slow:    340,   // sheets enter, page transitions
  xslow:   520,   // win celebration (expressive only)
};

// ── Common patterns ──────────────────────────────────────────────
export const motion = {
  // Sheet enter
  sheetEnter: { duration: dur.slow, easing: curve.kick },
  // Sheet exit (always shorter than enter)
  sheetExit:  { duration: 220, easing: curve.exit },
  // Page forward
  pageForward: { duration: 320, easing: curve.drive },
  // Stagger children (sheet, list)
  stagger: (index: number, base = 50) => index * base,
  // Odds flash
  oddsFlash: { flashIn: 0, flashOut: { duration: 300, easing: Easing.out(Easing.quad) } },
};
