Top bar + Sport tabs
fd.kick · 280ms · delay 0ms
The chrome arrives first, snapping into position from above. Fast — users have seen this before, so it should be nearly instant.

Featured game card
fd.kick · 340ms · delay 60ms
Slides up from slightly below. The 60ms delay after chrome means the structure lands before the content — users orient before reading.
translateY(8px) → 0, opacity 0→1
duration: 340ms, delay: 60ms
easing: bezier(0, 0, 0.06, 1)
3
Odds pills (featured card)
fd.release · 380ms · staggered 40ms
Odds pills are the most important data — they bounce in last, with a slight overshoot. The bounce draws the eye exactly where we want it. Stagger left-to-right signals reading order.
scale(0.8) → 1, opacity 0→1
duration: 380ms, delay: 120ms / 160ms
easing: bezier(0.34, 1.45, 0.64, 1)
4
Game list rows
fd.kick · 240ms · 50ms stagger each
Rows stagger up from below — like players walking onto the field one at a time. Shorter duration than the card (they're smaller, less weight). No bounce — these are functional list items.
translateY(6px) → 0, opacity 0→1
duration: 240ms, delays: 160/210/260ms
easing: bezier(0, 0, 0.06, 1)
5
Odds pill tap (any)
fd.snap · spring s:500 d:38
When a user taps an odds pill, it snaps to selected state instantly. Spring physics — no duration, just settles. The colour fill is instant; the scale snap is the spring. Try tapping a pill above.

scale: 0.94 → 1.04 → 1 (pressOut, spring)
bg: transparent → blue (50ms timing)

Market pill tap
fd.snap · spring · instant

fd.drive · 300ms · reverse axis
Going back reverses the direction — detail exits right, lobby returns from left. Slightly shorter (300ms vs 320ms) because return journeys should feel fast. Swipe-back gesture continues momentum.

duration: 300ms (shorter = faster return)
swipe: velocity continuation via spring

fd.kick · 340ms · ease-out aggressive

Input focus
fd.snap · instant border highlight
Border colour transition is 80ms linear — micro-interactions should feel immediate. The keyboard appearing is the main event; the border is supporting signal. Never animate focus states with a slow ease.
borderColor: transparent → blue
duration: 80ms, linear
(keyboard appearance handled by OS)
3
"Place Bet" press
fd.snap press-in · fd.release on confirm
Press-in: spring snap to 0.96 (anticipation). Release: spring back to 1. The snap happens in the button; the confirmation state appears in the sheet — two separate animations that feel like one continuous action.
pressIn: scale → 0.96, spring s:500 d:38
pressOut: scale → 1, spring s:300 d:20
then: trigger confirmation overlay
4
Confirmation overlay
fd.release · 440ms · earned expressive
The success state bounces in with overshoot — the first moment of expressiveness in this flow. The checkmark has its own spring delay (80ms), and the ring expands outward as secondary action. Three elements, one feeling.
card: scale(0.8)→1, bezier(0.34,1.45,0.64,1)
check: scale(0)→1, 80ms delay, same curve
ring: scale(1)→2.2, opacity 1→0, 500ms ease-out
5
Sheet dismisses
ease-in · 220ms · always faster than enter
Exit is always faster than enter — 220ms vs 340ms. Ease-in (accelerating) mirrors how objects naturally fall away. Backdrop fades in 180ms, concurrent. Swipe-down dismissal passes gesture velocity to spring.
translateY(0) → 100%
duration: 220ms
easing: Easing.in(Easing.cubic)

Live indicator
sine pulse · 1.2s · infinite
The live dot pulses at a breathing rhythm — sine easing so it never feels mechanical. Scale 1.0 → 0.8, opacity 1.0 → 0.4. Slow enough not to distract, present enough to signal live data. This never stops while in-game.

Odds moving up (favourable)
flash + micro-nudge · 80ms + 300ms
Green flash appears instantly, fades in 300ms. Value nudges 2px upward. Green = the line improved for the bettor. The nudge direction (up) reinforces the colour message — two signals, one glance. Tap any pill above to see it.

Odds moving down
red flash + downward nudge · same timing
Red flash + 2px downward nudge. Same duration as the up-flash — consistency is critical so users learn the pattern. The colour is the primary signal; the nudge is a secondary haptic-like cue for peripheral vision.
bg: transparent → red(0.18) → transparent
nudge: translateY(+2px)→0
(identical timing to up-flash)
4
Market suspended
opacity fade · 200ms · no spring
When a market suspends (e.g., near a scoring play), pills fade to disabled state. 200ms ease. No bounce, no drama — suspension is a signal to wait, not to panic. Re-activation fades back in the same way.
opacity: 1 → 0.4, color: blue → gray
duration: 200ms, easing: ease
cursor: not-allowed (interaction blocked)

1
Win screen enters
backdrop: 200ms · then card: fd.crowd 520ms
Backdrop fades first (200ms) — gives the user a moment to register something is happening. Then the card dunks in with fd.crowd: scale 0.7 → 1 with 1.8 overshoot. The extra bounce is the whole point.
backdrop: opacity 0→1, 200ms ease
card: scale(0.7)→1, bezier(0.34,1.8,0.64,1)
card delay: 150ms after backdrop
2
Win amount counts up
interpolate · 700ms · ease-out cubic
The number counts up from $0 to the win amount — building anticipation before the reveal. Ease-out means it slows as it approaches the final number, adding suspense at the end. Starts 300ms after the card enters.
interpolate(progress, [0,1], [0, WIN_AMOUNT])
progress: withTiming(1, {duration: 700
easing: Easing.out(Easing.cubic)})
delay: 300ms after card
3
Confetti
ease-in fall · staggered 0–500ms
~40 particles, each with a random start delay (0–500ms) so it rains, not floods. Ease-in on the Y-axis simulates gravity — they accelerate as they fall. Random rotation adds life. Keep it to 1 second total duration per piece.
y: -10px → screenHeight
duration: 900–1500ms (random per piece)
delay: random 0–500ms (the rain feel)
easing: Easing.in(Easing.quad)
4
CTA button enters
fd.kick · 280ms · delay 650ms
The CTA appears last — after the emotion has landed. If it appeared simultaneously with the win card, it would feel transactional. Waiting 650ms means the user has felt the win before being asked to do anything about it.
opacity 0→1, translateY(6px)→0
duration: 280ms
delay: 650ms after win card

Leg card enter
fd.kick · 280ms · translateY(12px→0)
Each new leg slides in from below, not from the side. Bottom-up entry grounds the card in physical space — it's being placed on a stack. Staggered when multiple enter at once, but single-leg adds are immediate.
withTiming(opacity, { duration: 280,

2
Leg count badge
fd.snap spring · s:500 d:38 · scale pulse
The badge bumps scale briefly on each new leg. Spring physics mean it settles without any explicit return animation. This makes the count feel reactive, not mechanical — like a scoreboard updating.
withSpring(scale, {
stiffness: 500, damping: 38
}) // from 1.2 → settles at 1.0
3
Combined odds counter
fd.snap · instant flash + 300ms fade
Odds update instantly — no count-up here, because the user needs to trust the number is accurate. A brief green color flash signals "this changed" and fades over 300ms. The number itself never animates.
// Flash color signal
color = '#00C87A'
withTiming(color, {
duration: 300,
easing: Easing.out(Easing.quad)
}) // → return to default
4
Potential payout count-up
ease-out cubic · 500ms delay:100ms
Unlike odds, the payout CAN count up — it's the emotional centrepiece. Use an ease-out so it arrives quickly and sits at the final number. A slight delay (100ms) after the odds update gives it a "reveal" beat.
// JS interpolate, not RN
const ease = 1 - Math.pow(1-t, 3);
current = ease * targetPayout;
// 500ms, 100ms delay
5
Odds Boost badge unlock
fd.crowd · bezier(0.34,1.8,0.64,1) · 520ms
Unlocking a boost at 3 legs is a peak moment — it deserves the most theatrical curve in the system. fd.crowd overshoots hard then settles, mimicking the crowd roar when something unexpected happens.
withTiming(scale, { duration: 520,
easing: Easing.bezier(0.34,1.8,0.64,1)
}) // scale(0.7) → scale(1.0)
withTiming(opacity, { duration: 200 })

Token reference

motionTokens.ts
import { Easing } from 'react-native-reanimated';

// ── Curves ──────────────────────────────────────────────────────
export const curve = {
  snap:    { stiffness: 500, damping: 38 },           // spring · buttons, toggles
  kick:    Easing.bezier(0.0, 0.0, 0.06, 1.0),        // enter screen
  drive:   Easing.bezier(0.4, 0.0, 0.10, 1.0),        // A→B within screen
  release: Easing.bezier(0.34, 1.45, 0.64, 1.0),      // success, confirm
  crowd:   Easing.bezier(0.34, 1.8,  0.64, 1.0),       // win, peak moments
  rebound: { stiffness: 220, damping: 7 },            // spring · expressive oscillation
  exit:    Easing.in(Easing.cubic),                     // any element leaving screen
};

// ── Duration ────────────────────────────────────────────────────
export const dur = {
  micro:   80,    // button press-in, focus highlight, odds flash
  fast:    150,   // tooltips, chips
  base:    250,   // toasts, tab content fade
  slow:    340,   // sheets enter, page transitions
  xslow:   520,   // win celebration (expressive only)
};

// ── Common patterns ──────────────────────────────────────────────
export const motion = {
  // Sheet enter
  sheetEnter: { duration: dur.slow, easing: curve.kick },
  // Sheet exit (always shorter than enter)
  sheetExit:  { duration: 220, easing: curve.exit },
  // Page forward
  pageForward: { duration: 320, easing: curve.drive },
  // Stagger children (sheet, list)
  stagger: (index: number, base = 50) => index * base,
  // Odds flash
  oddsFlash: { flashIn: 0, flashOut: { duration: 300, easing: Easing.out(Easing.quad) } },
};
